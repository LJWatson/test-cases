Scratch test case

This page is designed to get Safari's Reader mode to kick-in. The purpose is to test whether Safari's Reader mode supports the Cascading Style Sheets (CSS) speak-as property, as the VoiceOver screen reader seems to do.

Testing speak-as

The speak-as property is designed to let authors control the way speech output speaks certain types of content. For example, the digits value causes speech output to speak a number as a string of separate digits, instead of a whole large number. Here's an example:

This number should be spoken as digits: 01001110000, and this number should not: 01001110000.

Another example is the spell-out value. This is intended to cause synthetic speech to speak a word one character at a time. Here is another example:

This acronym should be spoken on letter at a time: NATO, but this time it should not: NATO.
