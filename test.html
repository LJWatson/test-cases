Scratch test page

This is a test case for a solution to a specific situation in which it is necessary to make a non-interactive element focusable in order to provide equivalent functionality to keyboard users (including screen reader users), that is otherwise available only to mouse users.

Making a non-interactive element focusable can cause confusion for screen reader users if they do not realise why a non-interactive element has received focus. Using aria-role-description is one potential technique that is being explored.

Use the Tab key to move focus onto the text below.
